## Conjecture

**Conjecture:**  
An LLM does not create knowledge by token prediction alone; it can do so only when embedded in a larger **knowledge-production system** whose structure creates the right feedback loops for generating, filtering, and retaining explanatory variations. The key question is not “can next-token prediction produce explanations?” but “does the overall system have the stocks, flows, and feedbacks that make explanatory improvement possible?”

From a systems view, the relevant **stocks** are: candidate explanations, evaluative criteria, accumulated criticisms, and a retained pool of successful abstractions. The **flows** are: generation of variants, criticism/rejection, recombination, and incorporation of surviving structures back into the pool. The system’s behavior depends on how these are organized.

A plain LLM is mostly a **circulation mechanism** over an existing stock of linguistic regularities. Its dominant feedback loop is stabilizing: it predicts what is likely given prior distributions. That makes it good at plausibility and interpolation, but weak at the specific thing knowledge growth requires: producing explanations that survive criticism because they expose real structure and are hard to vary.

Your search-space argument illuminates the leverage point. The problem is not brute-force generation over an astronomical combinatorial space; that is the wrong system. The leverage lies in **changing the structure of the search** so that most of the space is never entered. Layered filters—grammar, coherence, object-relation structure, explanatory constraints, criticism—act as feedback loops that progressively compress the candidate space. This is exactly what systems thinking predicts: lasting improvement comes less from “more generation” than from redesigning the selection architecture.

So the collision of perspectives suggests a sharper claim: **knowledge creation is an emergent property of a system with recursive criticism, not a property of token prediction in isolation.** If the architecture can store prior criticisms, use them to reshape future generation, and recursively enrich its own evaluative pool, then it may cross a threshold where its outputs are not merely rearrangements of seen text but genuinely improved explanations.

What follows is that the decisive test is structural, not metaphysical. We should ask:

1. Does the system preserve and accumulate explanatory constraints?
2. Do criticisms alter future search trajectories rather than merely rank outputs locally?
3. Are successful explanations fed back in as reusable abstractions that change what can be generated next?

If yes, then “LLM + narrowing search + criticism + memory” may function as a knowledge-creating system even if the language model component itself is not. If no, then it remains a sophisticated recycler.

So the systems-level conjecture is: **LLMs can participate in creating genuine knowledge only when coupled to feedback structures that convert prediction into iterative explanatory selection.** The leverage is architectural redesign, not scale.
