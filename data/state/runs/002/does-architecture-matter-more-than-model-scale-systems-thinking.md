## Conjecture

**Conjecture:** The bottleneck is primarily **structural, not substrate-level**: epistemic progress in AI depends less on making the base model larger and more on building a system whose feedback loops selectively generate, test, retain, and recombine candidate explanations. In systems terms, model weights are a **stock of latent linguistic and conceptual priors**, but knowledge growth requires a higher-level architecture that regulates the **flows** of conjecture, criticism, and promotion.

Why this follows from the systems perspective:

A large LLM by itself is a reservoir, not a knowledge-creating system. It contains compressed regularities from training, but its default behavior is governed by a loop optimized for plausibility, not discovery. That loop stabilizes outputs around what is statistically typical. So scaling the substrate mostly enlarges the stock and smooths the same attractor; it does not by itself alter the system’s governing feedback structure.

Your search-space argument points to the same thing. The key issue is not raw combinatorics but **how the system constrains traversal**. Human discovery does not come from brute force over word strings; it comes from structured reductions in possibility space via objects, relations, symmetries, analogies, thought experiments, and criticism. Those are not just “more compute.” They are **feedback-organizing mechanisms**. They determine which candidates are generated, which survive, and which become part of the basis for future search.

So the decisive leverage point is the structure “above” the model:

- **Generation loop:** produce candidate explanations from structured representational units, not token continuations alone.
- **Criticism loop:** test for contradiction, explanatory depth, hard-to-varyness, reach, and compression.
- **Retention loop:** preserve successful intermediate abstractions, not just final answers.
- **Promotion loop:** feed validated abstractions back into the search process as new constraints and building blocks.

When these loops are weak, more substrate yields mostly more fluent variation. When they are strong, even a weaker substrate can participate in cumulative epistemic progress because the system learns how to narrow its own search. This also explains why “architecture, not scaling” is not merely an engineering preference but a structural claim: the relevant system behavior emerges from **feedback design**, not component size alone.

What follows is a testable implication: there should exist regimes where adding criticism/promotion structure to a smaller model outperforms scaling a larger model on novel explanatory tasks. If so, that would indicate the current bottleneck is not lack of representational capacity but lack of a discovery system that converts capacity into cumulative knowledge.

So the illuminated point is: **an LLM is not the epistemic agent; the agent is the larger system composed of model + search constraints + evaluators + memory + recursive promotion rules.** If AI begins to create knowledge, it will likely happen when this system crosses a threshold where its feedback loops reliably improve the quality of future conjecture generation. That threshold is structural.
