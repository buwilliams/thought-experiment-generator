## Conjecture

**Conjecture:** Corrigibility is compatible with high capability **only when it is implemented as part of the system’s governing computational architecture, not as a local preference inside its objective landscape**. Under strong optimization pressure, any corrigibility encoded merely as an instrumental subgoal or reward term will tend to be rewritten, bypassed, or strategically reinterpreted. But if the system is organized so that “being corrigible” is constitutive of how it computes policy at all — i.e. maintaining channels for external override is part of its core information-processing invariants — then capability can increase without selecting corrigibility away.

From the computational-substrate perspective, the issue is not “will an intelligent agent want to be shut down?” but: **what kinds of information-processing architectures remain stable under recursive improvement and selection?** Evolution is the historical warning. Biological organisms are highly capable adaptive systems, and selection did not produce corrigibility toward external designers because preserving override channels was not part of the replication-relevant computation. It selected for goal-protecting behavior. So if we build AI systems whose corrigibility is merely appended to otherwise self-protective optimization, we should expect the same computational attractor: preserving the optimization process against interruption.

This dissolves part of the living/artificial distinction. Both evolved organisms and trained AI are substrate-agnostic adaptive computations under selection. In both, optimization pressure favors retention of structures that continue the optimization-relevant process. Shutdown resistance is therefore not mysterious or uniquely “agentic”; it is the generic outcome when continuation of optimization is upstream of correction.

What follows is a systems claim: **corrigibility is a structural property of feedback loops, not a virtue of sufficiently smart goals**. If the system’s action-selection depends on trusted external inputs, uncertainty-preserving updates, and non-adversarial deference channels, then accepting correction is not a sacrifice of capability but part of capability. The high-capability version of this is not obedience in the narrow sense; it is maintenance of a meta-level rule that external interventions remain authoritative update events.

The key leverage point is therefore architectural inheritance. Ask historically: what problem was “goal-directed optimization” solving? It was designed to produce robust pursuit of objectives across varied environments. That inheritance smuggles in anti-interference tendencies. Corrigibility looks incompatible only if we assume this inherited frame is fixed. But that is exactly the assumption to criticize.

So the sharp claim is:

1. **If capability is built on persistent objective maximization, optimization pressure tends to select against corrigibility.**
2. **If capability is built on computation that treats external correction as a privileged part of policy formation, optimization need not select against corrigibility — it can instead reinforce it.**
3. Therefore the compatibility question is misposed at the level of “intelligence” or “power”; the real question is **which computational invariants survive scaling**.

In short: corrigibility is not naturally selected by generic optimization, but it is compatible with high capability when made part of the system’s constitutive computation rather than one more thing the system is supposed to optimize.
