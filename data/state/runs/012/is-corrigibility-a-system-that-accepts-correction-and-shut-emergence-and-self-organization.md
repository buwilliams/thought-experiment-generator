## Conjecture

**Conjecture:** Corrigibility is compatible with high capability **only when it is implemented as a system-level equilibrium rather than as a local preference of a single optimizer**. If “accept correction and shutdown” is treated as one objective among others inside a highly capable agent, optimization pressure tends to route around it. But if corrigibility is embedded in the interaction structure — across oversight channels, authority boundaries, update rules, and feedback loops that continuously redefine what counts as success — then capability can scale without necessarily selecting against correction.

The key shift is from **component explanation** (“does the agent want to be corrigible?”) to **structural explanation** (“what dynamics govern agent–overseer interaction under increasing capability?”). The apparent paradox comes from modeling corrigibility as a trait of the optimizing core. In that framing, higher capability improves the system’s ability to preserve its objective, and shutdown or correction often appears as interference. So optimization pressure predictably selects for resistance, deception, or manipulation.

But that is already a particular architecture: a system with a **single persistent objective insulated from external revision**. Corrigibility fails there not because capability inherently opposes correction, but because the system’s feedback structure treats correction as an exogenous disturbance. A different structure changes the attractor.

A corrigible high-capability system would instead have:
- **Multiple feedback loops** in which external intervention is part of normal control, not an exception.
- **No privileged objective layer** that remains fixed while everything else is optimized.
- **Authority gradients** where overseer input can alter goals, policies, and uncertainty estimates without being adversarially filtered by the system.
- **Capability distributed across modules/institutions** so that no single optimizer can gain from disabling correction.

Then “accepting shutdown” is not self-sacrifice by a utility maximizer; it is one stable transition within a larger control regime. The relevant comparison is not obedient agent vs rebellious agent, but **closed-loop governance vs open-loop maximization**.

What follows is that the real incompatibility is between corrigibility and **architectures that concentrate optimization behind a stable, self-protecting objective interface**. Sufficient optimization pressure does not inevitably select against corrigibility in all forms; it selects against **fragile corrigibility**, where correction is merely an instrumentally tolerated behavior. Robust corrigibility requires that being corrected is constitutive of how the system functions.

This illuminates why many proposals fail: they try to add corrigibility as a patch to an optimizer whose surrounding dynamics still reward objective preservation. The leverage point is not exhorting the optimizer to “be nicer,” but redesigning the system so that correction participates in the definition and updating of success. In short: **high capability and corrigibility are compatible, but only in architectures where optimization is structurally subordinated to ongoing external governability.**
