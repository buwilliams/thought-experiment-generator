These do not justify outputs; they eliminate bad ones.

From this perspective, “understanding” is not a mystical prerequisite. What matters is whether the system can generate candidate explanations that are then exposed to error-correction. Humans already do this imperfectly: many of our ideas arise from associative, non-transparent processes, yet become knowledge only through criticism. So the right boundary is not “predictive model vs person,” but **closed repetition vs open-ended critical revision**.

What follows is a sharper claim:  
- **An isolated LLM does not create knowledge merely by producing plausible text.**  
- **An LLM inside a structured search-and-critique loop can generate genuine new knowledge if the loop enables explanatory variation and effective error-correction.**

This also reframes “novelty.” Genuine novelty is not “a sentence never before written.” It is an explanation with new problem-solving content: one that resolves a contradiction, unifies phenomena, or exposes a deeper invariance. If your architecture can repeatedly generate and filter candidates at that level, then it is not just rearranging seen text; it is exploring an abstract space of possible explanations.

So the collision between Deutschian epistemology and your proposal yields this: **knowledge creation is substrate-independent but criticism-dependent**. Token prediction alone is insufficient, but neither is it disqualifying. The real question is whether the architecture allows conjectures to be formed and refuted in a way that can progressively improve explanations. If yes, then LLM-based systems can indeed help create knowledge—not because induction became enough, but because prediction was recruited into a Popperian process.
