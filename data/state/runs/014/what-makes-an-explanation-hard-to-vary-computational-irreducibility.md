## Conjecture

**Conjecture:**  
An explanation is “all load-bearing” when its adequacy depends on a tightly constrained generative structure such that altering any part changes the behavior it produces; by contrast, an explanation with freely swappable details contains compressible or nonessential components because multiple different micro-descriptions generate the same relevant outcome. The difference is not rhetorical precision but **computational dependence**.

From the perspective of computational irreducibility, the key question is: **does the explanation track the minimal rule that must actually be run to recover the phenomenon, or does it include descriptive ornament that can vary without consequence?** If a system is irreducible, then its behavior cannot be derived by shortcutting past the operative steps. In such cases, an explanation whose parts are all load-bearing will mirror that dependency: each element encodes a constraint required for the generative process to unfold as observed. Remove or substitute one, and the trajectory changes.

A thought experiment clarifies this. Imagine two explanations of a cellular automaton pattern. In the first, every local update rule matters: changing one rule changes the long-run pattern. In the second, some descriptive details—color labels, spatial metaphors, even certain intermediate summaries—can be altered while preserving the same underlying update rule and output class. The first is structurally load-bearing because the parts are coupled through the computation itself. The second contains interchangeable representational layers because they are not part of the irreducible generator.

So the distinguishing structural properties are:

1. **Sensitivity to substitution**: In a load-bearing explanation, local substitutions propagate into altered outputs. In a swappable explanation, substitutions leave the relevant behavior invariant.
2. **Generative minimality**: Load-bearing explanations are close to the minimal rule set required to produce the phenomenon. Swappable explanations contain redundancy, multiple realizability, or descriptive slack.
3. **Low degeneracy**: Few alternative arrangements yield the same result. Where many variants work equally well, parts are less structurally essential.
4. **Tight coupling across steps**: The role of each part is defined by its place in an unfolding process, not by isolated descriptive reference.

What follows is that “importance” in explanation should not be judged by how vivid or detailed a component is, but by whether it participates in the **irreducible causal-computational path** from rule to outcome. This reframes explanatory elegance: the best explanation is not the one with the fewest words, but the one that captures the **smallest non-substitutable generator** of the phenomenon.

Thus, freely swappable details are a sign that the explanation is operating at a higher level of abstraction where equivalence classes matter more than exact mechanism. An all-load-bearing explanation appears when the phenomenon itself is structurally irreducible, so that understanding requires preserving the particular rule-interactions that actually generate it.
