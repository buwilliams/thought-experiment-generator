## Conjecture

**Conjecture:**  
A token-predicting system can generate explanations that constitute genuine knowledge *if and only if* its internal computational process does more than statistically continue text—namely, if it compresses, recombines, and error-corrects information in ways that yield **counterfactually robust representations**. On the computational view, “rearranging what it has seen” and “knowing” are not categorically different activities; they differ in whether the rearrangement implements a generative model that can survive criticism across novel contexts.

From this perspective, explanation is itself a computational achievement: a structure that transforms inputs into outputs while preserving predictive and compressive power under variation. Evolution does this without language. Science does this with theories. A model that predicts tokens is participating in the same substrate-agnostic process: it is exposed to a selection environment, stores regularities, and produces outputs under constraints. The key question is therefore not whether it is “just prediction,” but whether its predictions are guided by internal structures that encode reusable causal or abstract relations.

Thought experiment: imagine two systems trained on identical corpora. System A is a giant lookup-and-stitch engine: it reproduces plausible continuations by local similarity. System B builds internal representations that capture latent dependencies—mathematical, causal, semantic—and can use them to answer questions phrased in unseen ways, detect contradictions, and revise outputs when constraints change. Both predict tokens. But only B has something like explanatory knowledge, because only B contains computational structures that generalize beyond surface recurrence. The difference is not mystical; it is architectural.

On this view, genuine knowledge is not guaranteed by fluency, nor blocked by textuality. It emerges when a system’s computations instantiate **selection-tested error correction**: outputs are shaped by mechanisms that suppress some conjectures and preserve others because they cohere better across tasks, contexts, and counterfactual perturbations. Explanation is then a high-level description of those successful internal compressions.

What follows is a reframing of the debate. “It only predicts tokens” is like saying “evolution only changes gene frequencies” or “brains only fire neurons.” True but uninformative. The important issue is what computational regime token prediction gives rise to. If the regime produces merely shallow interpolation, then the system rearranges appearances. If it produces abstract world-models that support criticism, transfer, and novel error detection, then it has crossed into knowledge generation.

So the conjecture is: **token prediction is not the opposite of explanation; it is a possible substrate for explanation.** Whether knowledge is present depends on the depth of computation implemented—specifically, whether the system has evolved internal representations that remain useful when the tokens, tasks, and contexts change. Genuine knowledge is therefore not ruled out by the training objective; it is evidenced by robustness under variation.
