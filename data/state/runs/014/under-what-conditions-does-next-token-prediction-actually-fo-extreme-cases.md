## Conjecture

**Conjecture:**  
Next-token prediction forces a model to learn latent variables aligned with *real domain invariants* only in the limit where prediction across token sequences is impossible to compress without representing the hidden causes that remain stable across many surface realizations. In other words: prediction is sufficient only when the shortest successful predictive program must pass through variables that track the domain’s invariant structure.

Push to the extremes:

- **If the surface form is fully predictive**, then no invariants are needed. A giant n-gram table, retrieval system, or shortcut heuristic can solve the task. At this boundary, next-token prediction does **not** induce meaningful latent structure; it induces memorization or local correlation tracking.
- **If the next token is independent of any stable hidden cause**, then no invariant exists to be learned. Prediction cannot recover what is not there.
- **If many radically different surfaces are generated by the same hidden state**, and correct prediction requires generalizing across those surfaces, then compression pressure favors representing that shared hidden state. At this boundary, latent variables become unavoidable.

So the key condition is not “more data” or “bigger models” by itself, but a particular **causal-statistical geometry** of the data source:

1. **A persistent hidden state exists** that governs many observations/tokens.
2. **Surface variation is high**, so invariants cannot be captured by shallow token co-occurrence alone.
3. **Predictive success across contexts requires counterfactual stability**: the same hidden variable helps predict under many alternative phrasings, viewpoints, or local token histories.
4. **Model capacity and training pressure favor compression**, so representing one reusable hidden cause is cheaper than storing innumerable surface-specific rules.

At the limit, imagine infinitely many paraphrases of the same underlying situation. A predictor that does not encode the shared latent state must learn infinitely many separate mappings. A predictor that does encode the invariant can reuse one internal variable across all paraphrases. Thus, under enough variation, invariant-tracking is the only scalable solution.

But this also reveals the failure mode. If the training distribution is narrow, shortcuts remain finite and cheap. Then next-token prediction can achieve low loss while learning latents that are merely **predictive artifacts of the corpus**, not real invariants of the domain. The model learns what stays fixed **in the data-generating process it sees**, which may be social convention, formatting regularity, annotation bias, or discourse habit rather than ontology.

So the sharpened claim is:

> **Prediction is sufficient for learning real domain invariants only when the data distribution contains enough diverse transformations that preserve those invariants while scrambling superficial cues, and when successful compression across that diversity is cheaper via latent-state representation than via memorized local statistics.**

What the boundary reveals is this: next-token prediction does not inherently seek truth. It seeks reusable structure. It learns real invariants exactly when reality’s invariants are the most reusable structure available.
