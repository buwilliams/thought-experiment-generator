# Thought Experiment Generator
## Design Document
*Personal reference — work in progress*

---

## Abstract

A 1,000-word thought experiment drawn from a 50,000-word vocabulary has a theoretical search space of 50,000^1,000 ≈ **10^4,699** possible combinations — larger than the number of atoms in the observable universe. This space is intractable by any exhaustive method. And yet Einstein, drawing on all the background knowledge accumulated throughout his life up to that point, navigated it to produce special relativity. His starting question, *what would a frozen light wave look like?*, was not a bolt from nowhere. It was an informed guess made possible by everything he already knew.

The Thought Experiment Generator is a system for making informed guesses. It takes background knowledge about a topic and collides it with sentences generated from random words — using the randomness as a forcing function to reach territory that pure topic knowledge would never find on its own.

The process is flat and parallel: generate a pool of background sentences, generate a pool of random-word-seeded sentences, combine them into thought experiments, criticize each one. No tree. No branches. No filtering. Every generated thought experiment survives to the scoring stage and is judged on its own merits by the principles of fallibilism.

---

## Architecture

### Core Concept

The generator works in three phases: **Create**, **Combine**, and **Criticize**.

The novelty in this system comes from the **randomness algorithm, not the LLM**. Random words force conceptual collisions that topic-specific prompting cannot reach. The LLM's job is to reason faithfully from those forced inputs — to follow the logic of the collision, not to invent. The background sentences provide enough grounding that the result remains coherent and relevant. Novelty emerges from the collision itself.

This is how Einstein used thought experiments: not by trying to be creative, but by taking a strange premise seriously and following it wherever it led.

---

### Phase 1: Create

Three files are generated and cached on first run:

**`background.txt`** — A pool of sentences about the topic, biased toward anomalies, unresolved tensions, edge cases, and open questions rather than established textbook facts. Generated by a single LLM call.

**`words.txt`** — Lines of random words sampled from `data/words_alpha.txt`. Each line has `--words` words (default: 5). The number of lines equals `--pool-size` (default: 50).

**`generated.txt`** — Each line of `words.txt` converted into a sentence that preserves the strangeness and tension of the word combination. The LLM is explicitly told not to normalize the words into familiar claims. Generated by a single LLM call processing all lines at once.

All three files are reused on subsequent runs unless `--fresh` is passed.

---

### Phase 2: Combine

For each thought experiment (1 through `--experiments`):

1. Pick `--background` random sentences from `background.txt` (default: 3)
2. Pick `--generated` random sentences from `generated.txt` (default: 2)
3. Send to LLM: treat all sentences as axioms — assume they are simultaneously true — and reason out what that implies about the world

The LLM acts as a reasoner, not a storyteller. It does not invent novelty or try to be creative. It follows the logic of the forced collision.

Output: `experiments/NNN-experiment.txt`

All experiments run concurrently. Each is independently seeded from the pools.

---

### Phase 3: Criticize

Each thought experiment is scored independently using the principles of fallibilism:

- **Reach** (0.0–1.0) — does it propose something no prior work has proposed, breaking into genuinely new explanatory territory?
- **Novelty** (0.0–1.0) — does it reframe or connect existing knowledge in a non-obvious way?
- **Falsifiable** (0.0–1.0) — is it testable or disprovable?

Reach and Novelty measure different things: Reach is the harder bar (genuinely new territory beyond the corpus of human knowledge); Novelty is reframing within known knowledge. Each score is accompanied by a brief explanation (≤20 words). A short overall explanation (≤20 words) is also generated.

**Total score** = reach + novelty + falsifiable (max: 3.0)

Output: `experiments/NNN-experiment-criticize.json`

Criticism runs at low temperature (0.2) for consistent scoring. Generation runs at the configured temperature (default: 1.0).

---

### Phase 4: Results

Results are sorted by total score. A summary is written to `summary.md` containing:

- A ranked table: experiment name, reach, novelty, falsifiable, total score, with links to experiment and critique files
- Top 5 thought experiments with a 20-word LLM-generated summary each

## LLM Prompt Templates

All prompts use a system/user split. Instructions go in the system prompt (static); dynamic content goes in the user message.

### Prompt 1 — Background Generation
*Temperature: configured (default 1.0)*

**System:** Generate sentences as a plain newline-separated list. No numbering, no formatting, no preamble.

**User:** Given the following topic context, write {pool_size} sentences as a newline-separated flat list with no formatting. Prefer anomalies, unresolved tensions, edge cases, and open questions over established textbook facts. Draw directly from the specifics of the context provided.

Topic context: {topic}

---

### Prompt 2 — Words to Sentences
*Temperature: configured (default 1.0)*

**System:** Convert each line of words into one sentence. Return exactly one sentence per input line, in the same order. No numbering, no formatting, no preamble.

**User:** Turn each line into a sentence that uses all the words on that line. The sentence does not need to make conventional sense — preserve the strangeness and tension of the word combination. Do not normalize it into a familiar claim. Lines: {word_lines}

---

### Prompt 3 — Thought Experiment Generation
*Temperature: configured (default 1.0)*

**System:** You are a reasoner, not a storyteller. Your job is to take the provided sentences as axioms — assume all of them are simultaneously true — and reason out what that implies about the world. Write a thought experiment in 500 words or fewer that follows strictly from those premises. Do not invent novelty. Do not be creative for its own sake. Just follow the logic of the collision. No preamble.

**User:** Premises: {background_sentences} {generated_sentences}

---

### Prompt 4 — Criticism
*Temperature: 0.2 (deterministic)*

**System:** Evaluate thought experiments using the principles of fallibilism. Return only valid JSON with no preamble or markdown.

**User:** {thought_experiment}

Using the principles of fallibilism decide:
- Reach: Does the implication of this thought experiment propose something no prior work has proposed — does it break into genuinely new explanatory territory, score 0.0–1.0?
- Reach: Why (in 20 words or less)?
- Novelty: Does it reframe or connect existing knowledge in a non-obvious way, score 0.0–1.0?
- Novelty: Why (in 20 words or less)?
- Falsifiable: Is the thought experiment falsifiable, score 0.0–1.0?
- Falsifiable: Why (in 20 words or less)?
- In less than 20 words, what does it explain or mean?

```json
{
    "reach": 0.0,
    "reach_why": "",
    "novelty": 0.0,
    "novelty_why": "",
    "falsifiable": 0.0,
    "falsifiable_why": "",
    "explanation": ""
}
```

---

### Prompt 5 — Top 5 Summary
*Temperature: 0.3*

**System:** Summarize the following thought experiment and its critique in 20 words or fewer. Return only the summary with no preamble.

**User:** {thought_experiment_text} {critique_json}

## CLI Parameters

```
--experiments    Number of thought experiments to generate (default: 20)
--pool-size      Total sentences in the background and generated pools (default: 50)
--background     Background sentences to use per thought experiment (default: 3)
--generated      Generated sentences to use per thought experiment (default: 2)
--words          Random words per line in words.txt (default: 5)
--temperature    LLM temperature for generation (default: 1.0)
--max-concurrent Max concurrent LLM calls (default: 5)
--provider       anthropic | anthropic-token | openai (default: anthropic)
--model          Model name (default: claude-sonnet-4-6)
--topic-file     Path to a file to use as the topic (supports long context)
--fresh          Clear cache and start over
--read           Show cached summary without running
```

## Output Files

All output lives in `data/cache/[topic-hash]/`:

```
background.txt                             — sentence pool about the topic
words.txt                                  — random word groups
generated.txt                              — sentences generated from word groups
experiments/001-experiment.txt             — thought experiment #1
experiments/001-experiment-criticize.json  — scores for experiment #1
...
summary.md                                 — ranked table + top 5 summaries
```

The hash is derived from the topic string (lowercased, trimmed) so the same topic always maps to the same directory. Changing parameters does not invalidate the cache — use `--fresh` to start over.

---

## Resumability

Each phase is independently resumable:
- If `background.txt` exists, background generation is skipped
- If `words.txt` exists, word selection is skipped
- If `generated.txt` exists, sentence generation is skipped
- If `experiments/NNN-experiment.txt` exists, that experiment's generation is skipped
- If `experiments/NNN-experiment-criticize.json` exists, that experiment's criticism is skipped

Interrupted runs pick up where they left off.

## Glossary

| Term | Definition |
|---|---|
| **Background pool** | Sentences about the topic biased toward anomalies and open questions, LLM-generated |
| **Generated pool** | Sentences built from random words, preserving strangeness |
| **Words pool** | Random word groups sampled from `words_alpha.txt` |
| **Pool size** | Total sentences in the background and generated pools |
| **Reach** | Whether a thought experiment breaks beyond the existing corpus of human knowledge into new territory |
| **Novelty** | Whether a thought experiment reframes or connects existing knowledge in a non-obvious way |
| **Falsifiable** | Whether a thought experiment is testable or disprovable |
| **Total score** | Sum of reach + novelty + falsifiable (max 3.0) |
| **Fallibilism** | The epistemological principle that knowledge grows through conjecture and criticism (Popper/Deutsch) |
